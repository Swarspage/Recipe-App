## 📡 BACKEND API ROUTES

### Auth `/api/auth`
- POST `/register` — hash password, set session, return user (no password)
- POST `/login` — validate, set session
- POST `/logout` — destroy session
- GET `/check` — return current user from session

### Recipes `/api/recipes`
- GET `/` — all recipes (limit 100)
- GET `/:id` — single recipe
- On server start: run `seedRecipes()` — if collection empty, seed from `recipes.json`

### User `/api/user` (all protected — check req.session.userId)
- GET `/saved` — populated saved recipes
- POST `/saved` — add recipeId to saved
- DELETE `/saved/:id` — remove from saved
- GET `/profile` — user info + savedCount + joinDate
- GET `/planner` — get weekly planner
- POST `/planner` — update weekly planner `{ day, recipeId }`

### AI `/api/ai` (protected)
- POST `/substitute` — body: `{ ingredient, recipeContext }` → call Groq API → return substitute suggestion

## ⭐ THE CROWN JEWEL — PANTRY MATCHING ALGORITHM

**`/pantry` page — this is the core differentiator.**

### Layout: Two-panel split
**Left (40%):** Ingredient selector
**Right (60%):** Matched recipe cards, animated

### Left Panel Logic:
1. On mount, fetch ALL recipes
2. Extract every unique ingredient across all recipes → build a categorized map:
   ```
   Vegetables: [Onion, Tomato, Potato, Garlic...]
   Grains & Staples: [Rice, Atta, Bread, Maggi...]
   Dairy & Eggs: [Egg, Milk, Curd, Butter, Paneer...]
   Spices: [Cumin, Turmeric, Chili, Coriander...]
   Proteins: [Chicken, Dal, Paneer, Soya...]
   Condiments: [Oil, Salt, Soy Sauce, Ketchup...]
   ```
3. Search bar to filter ingredient chips
4. **Voice Input button** — uses Web Speech API to listen and auto-toggle matching ingredient chips
5. IngredientChip component: pill shape, on unselected = cream background + green border, on selected = forest green background + gold checkmark + white text, Framer Motion spring animation on toggle

### Right Panel — Matching Algorithm:
```js
const matchedRecipes = useMemo(() => {
  if (selectedIngredients.size === 0) return [];
  
  return recipes
    .map(recipe => {
      const recipeIngredients = recipe.ingredients.map(i => i.toLowerCase());
      const selected = [...selectedIngredients].map(i => i.toLowerCase());
      const matched = selected.filter(i => 
        recipeIngredients.some(ri => ri.includes(i) || i.includes(ri))
      );
      const missing = recipeIngredients.filter(ri => 
        !selected.some(i => ri.includes(i) || i.includes(ri))
      );
      const matchScore = Math.round((matched.length / recipeIngredients.length) * 100);
      return { ...recipe, matchScore, matchedIngredients: matched, missingIngredients: missing };
    })
    .filter(r => r.matchScore > 0)
    .sort((a, b) => {
      if (b.matchScore !== a.matchScore) return b.matchScore - a.matchScore;
      return a.time_minutes - b.time_minutes; // tie-break: faster first
    });
}, [selectedIngredients, recipes]);
```

### RecipeCard shows:
- Match score badge: 
  - 100% → forest green badge "Perfect Match ✓"
  - 75-99% → gold badge "X% Match"
  - 50-74% → cream/outline badge "X% Match"
  - <50% → subtle gray badge
- Missing ingredients count: "Missing 2 ingredients"
- Time + difficulty chips
- Framer Motion: `AnimatePresence` with `layout` prop so cards smoothly reorder as you toggle chips
